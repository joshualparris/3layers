# Narrative And Canon Bible

## Canon Governance

Every fact must have:

- Layer (`L1`, `L2`, `L3`)
- Canon status
- Source reference (file or adjudication note)

## Active Canon Baseline

- Campaign: Storm King's Thunder, Chapter 2 context active.
- Party continuity includes Alphie, Wren, Dorrin, and Kella with special NPC pairings in Goldenfields context.
- Key mission thread: deliver death news and Rillix to Miros before broader combat escalation.
- Major unresolved: Alphie's ruleset/build alignment (2014 Divination vs 2024 Evocation representation).

## Source Hierarchy

1. Sourcebook and source-canon references.
2. Explicit table rulings recorded in handover/checkpoint docs.
3. App-recorded table-canon updates.
4. Inferred continuity notes (must remain marked inferred).

## Conflict Handling Protocol

When two sources disagree:

1. Record both claims.
2. Set status to `unresolved`.
3. Add required adjudicator (e.g., Marcus + Josh).
4. Do not auto-promote to `table-canon` until resolved.

## DM-Private Handling

- Facts tagged `DM-private` are stored but isolated.
- Any player-visible output must exclude DM-private by default.
- QA must include leak tests for DM-private fields.

## Narrative Integrity Rules

- Character voice consistency is retained from handovers.
- Emotional arcs are documented without forcing player outcomes.
- Tactical summaries cannot rewrite established chronology.

## Canon Maintenance Cadence

- Per session: validate all changed quests/notes.
- Weekly: run contradiction review.
- Milestone release: full canon audit against source docs and checkpoint files.
